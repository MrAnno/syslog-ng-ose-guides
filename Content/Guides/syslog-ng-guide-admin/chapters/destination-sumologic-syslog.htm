sumologic-syslog()

The sumologic-syslog() and sumologic-http() destinations send log messages to sumo logic, a cloud-based log management and security analytics service. You can send log messages over TCP, or encrypted with TLS.

Sending data using the sumologic-syslog destination
Example: Sending data using the sumologic-syslog() destination

The following example illustrates how the system() source to your Loggly account.

log {
destination

{ sumologic-syslog(token("USER-TOKEN-AS-PROVIDED-BY-sumologic") deployment("ENDPOINT") tls(peer-verify(required-trusted) ca-dir('/etc/syslog-ng/ca.d')) ); }
;
};

To use the loggly() driver, the scl.conf file must be included in your configuration:

@include "scl.conf"

Example: Sending data using the sumologic-syslog() destination

The following example illustrates how the system() source to your Loggly account.

log {
					source

					{ system(); }
					;
					destination

					{ sumologic-syslog(token("USER-TOKEN-AS-PROVIDED-BY-sumologic") deployment("ENDPOINT") tls(peer-verify(required-trusted) ca-dir('/etc/syslog-ng/ca.d')) template("$(format-json --scope all-nv-pairs)")); ); }
					;
				};
