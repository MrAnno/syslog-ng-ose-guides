system() source options

The system() driver has the following options:

exclude-kmsg()
Type: yes | no
Default:

Description: This option makes it possible to avoid duplicate collection of kernel logs or errors in kernel log collection, (for example, in scenarios where the log management on the host system and the containerized solution are collecting the kernel logs simultaneously). When set to yes, will omit kernel logs on platforms where they are available separatedly. Currently these platforms are the following:

  • Linux without systemd

  • FreeBSD
