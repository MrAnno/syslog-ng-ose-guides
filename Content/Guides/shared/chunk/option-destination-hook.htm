hook-commands()

Description: This option makes it possible to execute external programs when they are initialized or torn down. The hook-commands() can be used for both source and destination drivers.

setup()
Type: string
Default: N/A

Description: Defines the external program that the hook-commands() executes automatically as is started/stopped.

teardown()

Type: string
Default: N/A

Description: Defines the external program that the hook-commands() stops automatically as is started/stopped.

Example: Example for using the hook-commands()

In the following example, the hook-commands() opens an iptables port automatically as is started/stopped.

The assumption in this example is that the LOGCHAIN chain is part of a larger ruleset that routes traffic to it. Whenever the created rule is there, packets can flow, otherwise the port is closed.

 

source {
   network(transport(udp)
	hook-commands(
          setup("iptables -I LOGCHAIN 1 -p udp --dport 514 -j ACCEPT")
          teardown("iptables -D LOGCHAIN 1")
        )
     );
};
