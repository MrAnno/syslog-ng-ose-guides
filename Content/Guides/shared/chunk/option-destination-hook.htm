hook-commands()

Description: This option makes it possible to execute external programs when the relevant driver is initialized or torn down. The hook-commands() can be used with all source and destination drivers with the exception of the usertty() driver.

setup()
Type: string
Default: N/A

Description: Defines the external program that the hook-commands() executes as reloads.

shutdown()
Type: string
Default: N/A

Description: Defines the external program that the hook-commands() executes as stops.
startup()
Type: string
Default: N/A

Description: Defines the external program that the hook-commands() executes as starts.

teardown()
Type: string
Default: N/A

Description: Defines the external program that the hook-commands() executes after reloads.
Example: Example for using the hook-commands()

In the following example, the hook-commands() is used with the network() driver and it opens an iptables port automatically as is started/stopped.

The assumption in this example is that the LOGCHAIN chain is part of a larger ruleset that routes traffic to it. Whenever the created rule is there, packets can flow, otherwise the port is closed.

source {
   network(transport(udp)
	hook-commands(
          startup("iptables -I LOGCHAIN 1 -p udp --dport 514 -j ACCEPT")
          shutdown("iptables -D LOGCHAIN 1")
        )
     );
};
